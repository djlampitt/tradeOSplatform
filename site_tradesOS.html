Architecture

One event bus. Every model reacts.

The DataSpine is a typed, pub/sub event bus. Models subscribe to topics and publish results — no direct model-to-model coupling, no integration work, no glue code.

Capabilities

Built for the shop floor

Every feature addresses a real operational gap — not a dashboard metric.

🔍

Root-Cause in Real Time

WeldEye doesn't just flag a defect. It classifies operator vs. process vs. material in the same call, so the right fix goes to the right person — not just the nearest supervisor.

🔄

Feedback Loops That Close

A defect traced to bad material automatically flags the supplier lot, checks genealogy across all jobs, and routes an estimated cost exposure for approval — without a human relay.

👷

Crew Dispatch Without the Spreadsheet

CrewMatch matches required skill codes, certifications, and clearances (cGMP, DoD, cleanroom) against your live workforce — and explains every tradeoff in plain language.

🎓

Training Driven by Production Data

SkillForge assigns micro-modules and hands-on remediations based on actual defect events, not static curriculum cycles — so training is always calibrated to what's actually going wrong.

💰

Cost Gates Before You Commit

Every work order gets a cost estimate and an approval tier (routine / elevated / capital) before a tech touches the floor. PilotCheck enforces who can sign what, with a full audit trail.

📋

Pre-Project Feasibility, Not Gut Feel

ProjectPilot evaluates 3 approach options — timeline, cost, risk, sign-off level — before work commences, replacing the gut-check with a reasoned, AI-assisted recommendation.

Technology

Built on Claude. Built for the floor.

TradesOS uses Claude Sonnet for high-volume, latency-sensitive triage — WeldEye classifies defects, FaultLine generates work orders, CrewMatch resolves skill matches, and ChainSight traces genealogy, all in parallel event streams.

Claude Opus handles the higher-stakes reasoning: SkillForge designs remediation paths, PilotCheck evaluates AI-assisted decisions against governance rules, and ProjectPilot compares approach tradeoffs for capital projects.

Every model has an offline fallback — deterministic rule-based logic that produces the same output format as the live path. No API key required for development, testing, or air-gapped deployments. The event bus runs in-process with zero infrastructure dependencies.

The pub/sub topology means any model can be swapped, versioned, or replaced without touching the others. The spine is the only contract.

data_spine.py
# DataSpine — pub/sub event bus class DataSpine: def subscribe(self, topic: str, handler: Callable) -> None: self._subs[topic].append(handler)   def publish(self, topic: str, payload: dict) -> None: self._events.append({ "topic": topic, "payload": payload, "ts": _now() }) for h in self._subs[topic]: h(payload)
